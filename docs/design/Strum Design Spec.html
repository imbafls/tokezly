Design Specification · Local-first dictation

Strum.

A high-performance, desktop-native transcriber for engineers. Speak anywhere, get clean text everywhere. At its core it's a fast local transcriber — speech to text, pasted anywhere, no AI required. Turn the AI rewrite on and it polishes what you said, fully on-device. At runtime it's almost nothing on screen: a tiny translucent capsule, a small popup, then gone. The only full window is Settings.

02

Footprint & Surfaces

Two surfaces, wildly different sizes. The runtime is a tiny translucent capsule — about 46px tall — that materializes on a global shortcut, shows a waveform while you talk, swaps to a small popup as text streams, flashes a paste toast, then disappears. The only full window is Settings (solid dark, sidebar nav). No taskbar entry, no focus steal.

◆ Anatomy — the listening capsule ~ 230 × 46 px
ARecording dotAlert red, pulses while the mic is open. The one always-on signal that you're live.
BWaveformLive mic amplitude — the only motion. Replaces a spinner: you can see it's hearing you.
CTimerElapsed seconds in Cascadia mono. That's the entire listening UI.
→Then it grows, brieflyWhen text arrives the capsule becomes a small popup (a spinner + engine hint + streaming text), then a paste toast, then nothing.
Footprint discipline: nothing persistent. Strum is invisible until you summon it and gone the moment it has pasted. The capsule never docks or takes a window slot.
03

The Runtime, Live

The real component, fully interactive. Hit Dictate to watch the capsule listen → a popup stream the transcript → a paste toast. Hit Refine to run a Prompt rewrite. Or scrub any single state with the switcher. Everything is one React component driven by props — the source of truth for every frame in this spec.

Try: press Dictate and let the wave settle into a streaming popup and a paste toast. Then press Refine — the same speech becomes a commit message. Use the state chips to freeze any frame.
04

Transcribe → Clean → Prompt

Three behaviors, one base. With AI off, Strum is a plain transcriber — Ctrl + Space dictates and pastes verbatim. With AI on, the same dictation is auto-cleaned — fillers and repeats gone, self-corrections applied, grammar fixed. And Prompt — Ctrl + Shift + Space or typing / — runs an explicit instruction through the command popup. The AI layer is a single toggle in Settings, never clutter on the capsule.

AI Clean — auto-polish (default when AI on) streaming popup

Speak a messy sentence → Strum pastes the polished version: fillers and repeats gone, self-corrections applied, grammar fixed. Turn AI off and this step is skipped — you get the verbatim transcript instead.

/ Prompt — command popup command field

A / opens the instruction field with recent + suggested rewrites. Your dictation is the payload; the instruction shapes it. Enter runs it, Esc cancels.

05

Status & the AI Thinking State

Feedback is one quiet row inside the popup: a small spinner, the phase, the engine, and live latency in milliseconds. No modal, no full-screen loader. During a rewrite the refined text streams token-by-token while the ring spins and the millisecond counter climbs — you read the answer forming instead of watching a spinner.

● Listening capsule · waveform

The capsule shows a red dot, a live waveform, and a timer. That's it — nothing to read yet, so nothing expands. The motion is the feedback.

▸ Transcribing popup · streaming

A small popup opens. Text streams in Cascadia mono so nothing reflows. The status row carries the spinner, engine, throughput and live latency.

✦ Refining popup · AI rewrite

What it heard stays dimmed above; the refined output streams below with an accent rail and a Sparkle. The status row switches to the refinement model and its latency.

The status row — anatomy

◜◞ ring  phase · engine · throughput ↳ latency ms

Latency is first-class — engineers trust tools that show their cost. The number updates live during the call and freezes on the final figure once text settles.

Thinking, without the noise

  • No full-screen overlay; no shimmer over already-good text.
  • The spinner is ~15px and sits inline with the status text.
  • Inserted tokens flash accent briefly, then settle — a subtle diff trail.
  • If a rewrite stalls > 4s, the row shows a cancel hint (Esc), never an endless spinner.
  • After paste, the toast auto-dismisses in ~1.2s.
06

System-Wide Shortcuts

Strum is a keyboard tool first — fired from inside your editor, terminal, or browser without ever clicking. The idle capsule shows its binding; every key here is a global hotkey, remappable in Settings → Shortcuts. Glyphs follow the host OS — Windows shown.

07

Engine Status & Extensibility

Strum is engine-agnostic — Parakeet, Whisper, Moonshine and more swap underneath. The runtime never shows this complexity: at most a tiny status dot + model name in a popup header. Everything is managed in Settings → Engines, where ASR engines and the optional refinement model sit in clean settings cards — accuracy/speed meters, badges for Active/Local, local vs cloud made explicit.

◷ Runtime hint — engine in the header zero clutter

The only place the active engine surfaces at runtime: a 6px dot + model name in the streaming popup's header. Green = connected. If an engine drops, it turns alert red and an inline banner offers a fix.

08

UI across configurations

Strum scales its UI with capability — it never restructures. The listening capsule is identical whether or not AI is on; the only thing that changes is the result stage. Plain STT pastes silently behind a toast. With AI on, one small popup appears because there's a transform worth watching. Same vocabulary, one extra surface.

STT onlyAI off2 surfaces
1 Idle shared
2 Listening shared
3 Paste — verbatim
Capsule → toast. No popup — nothing to read, so nothing opens.
STT + AIAI on3 surfaces
1 Idle identical
2 Listening identical
+ Clean — the AI transform
3 Paste — cleaned
Capsule → one popup (the transform) → toast. Prompt / is available too.
The rule: turning AI on adds exactly one surface — the popup that shows the rewrite forming — and unlocks the / command. Everything else (the trigger pill, the listening capsule, the paste toast) is byte-for-byte the same. Features change what Strum does, never how much window it throws at you.
